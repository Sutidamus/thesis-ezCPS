EZ-CPS

What is this?

Students take CSSE 304 Programming Language Concepts are introduced to a paradigm of writing code called continuation passing style (CPS). CPS requires that all code be written such that there is no need to ever return back to a caller function. For this to be possible, two things must be true.


  1. All calls to substantial procedures/functions must be in tail position
  2. Every substantial procedure must take a continuation (in this case a lambda) that states what to do next

This site will test students' abilities to solve problems about CPS transformations (transforming non-CPS procedures to CPS). With your help, we hope to gain a better understanding of what problems students struggle with when designing CPS code & how to best address them.

NOTE: For this exercise, continuations are represented using LAMBDAs, and they are created using make-k and applied using apply-k


;;=======================================
;; Definitions
;;---------------------------------------
;; These procedures are already defined, so you should NOT define them.

(define apply-k (lambda (k v) (k v)))
(define make-k (lambda (k) k))

;;=======================================
;; Example Use
;;---------------------------------------
(define sampleProc (make-k (lambda (v) (list v))) )
(apply-k sampleProc 2) ;; => (2)
